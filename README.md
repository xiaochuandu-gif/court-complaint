# 一键投诉法院不立案不下裁定 v2.0

针对法院立案庭**不立案且不下不予立案裁定**的违法行为，一键自动向多个受理法官违法违纪的平台进行投诉。

## v2.0 新增功能

- **自动登录**：基于 Selenium 浏览器自动化，自动登录各投诉平台
- **自动注册**：使用预设手机号和邮箱自动注册新账号
- **账号管理**：本地加密保存各平台的用户名和密码
- **手动接管**：遇到验证码、人机验证等问题时，弹窗提示用户手动操作
- **登录日志**：实时显示自动登录进度和状态
- **双模式**：支持"自动登录模式"和"传统模式（仅打开网页）"

## 功能特点

- **14 个投诉渠道**，覆盖法院、检察院、纪检监察、人大信访、信访局、司法部、政法委等
- **31 个省份**自动匹配省人大信访和省信访局网址
- **一键操作**：自动复制投诉内容到剪贴板，并打开所有选中平台的投诉页面
- **内置投诉模板**，包含法律依据，可直接使用或自行修改
- **草稿保存/加载**，支持文件导入导出，投诉历史自动记录

## 快速开始

```bash
# 1. 克隆仓库
git clone https://github.com/your-username/court-complaint.git
cd court-complaint

# 2. 基础运行（传统模式，无需额外依赖）
python 一键投诉.py

# 3. 安装自动登录依赖（可选）
pip install selenium webdriver-manager

# 4. 运行（支持自动登录）
python 一键投诉.py
```

## 使用步骤

### 传统模式（无需额外依赖）

1. **选择省份**：在顶部下拉框选择所在省份
2. **撰写投诉内容**：在左侧文本框中撰写投诉材料（可点击"加载模板"使用预设模板）
3. **选择投诉平台**：在右侧勾选要投诉的平台（默认全选）
4. **一键投诉**：点击"传统模式（仅打开网页）"按钮
5. **粘贴提交**：在每个投诉页面中按 Ctrl+V 粘贴投诉内容并提交

### 自动登录模式

1. **安装依赖**：`pip install selenium webdriver-manager`
2. **配置账号**：切换到"账号管理"标签页，设置默认手机号、邮箱和密码
3. **设置平台账号**：为各平台设置登录账号（或使用"批量设置"一键填充）
4. **撰写投诉内容**：在"投诉内容"标签页撰写材料
5. **一键投诉**：点击"一键投诉（自动登录）"按钮
6. **手动接管**：遇到验证码等问题时，在弹出的浏览器中手动操作，完成后点击"继续"
7. **粘贴提交**：在各投诉页面粘贴内容并提交

## 手动接管机制

当自动登录遇到以下情况时，程序会暂停自动化并弹出提示窗口：

- **验证码**：需要输入图片验证码
- **短信验证码**：需要输入手机收到的验证码
- **人机验证**：滑块验证、点选验证等
- **实名认证**：需要上传身份证等材料
- **其他异常**：页面结构变化、网络问题等

用户在浏览器中完成操作后，点击"已完成，继续"即可恢复自动化流程。也可以点击"跳过此平台"跳过当前平台。

## 法律依据

- 《中华人民共和国民事诉讼法》第一百二十六条
- 《最高人民法院关于人民法院登记立案若干问题的规定》
- 《人民法院工作人员处分条例》

## 注意事项

1. 投诉内容请实事求是，如实反映情况
2. 建议保留好起诉材料的递交凭证（如收据、EMS回执等）
3. 部分投诉平台需要实名注册后才能提交
4. 投诉后请关注各平台的回复和处理进度
5. 账号密码以 Base64 编码保存在本地 `~/.court_complaint/` 目录
6. 自动登录功能依赖各平台页面结构，如平台改版可能需要更新适配器
